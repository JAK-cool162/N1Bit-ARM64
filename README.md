# N1Bit-ARM64

An ultra-low-power, maximum-speed **1-Bit AI Engine & Development Framework** (BitNet b1.58 style) written entirely from scratch in Python and NumPy. This engine is specifically optimized for **ARM64 / Mobile environments** (like Termux on Android), with full dynamic fallbacks to accelerate with PyTorch, CUDA, and Vulkan when run in PC or GPU-enabled environments.

---

## 🚀 Key Upgrades

### 1. 🎨 Touch-Optimized Web UI Dashboard (`app.py`)
In addition to the console-based CLI, a spectacular, modern web-based graphical dashboard is now built into your project. 
- Designed with **Tailwind CSS** and **FontAwesome**, optimized for mobile phone touch screens.
- **ChatGPT-Style Causal Chat:** Featuring real-time response generation, scroll logs, and message indicators.
- **🧠 Real-Time Token Thoughts Panel:** For every single word generated, the dashboard displays the **Top 5 token alternatives** (what words the AI was choosing between) with glowing probability bars!
- **⚡ Active Neuron State Grid:** Renders an interactive grid of **64 glowing neuron dots** showing live positive (`+1`) vs. negative (`-1`) binary states inside your active layers!
- **🏋️ Live Training Console:** Toggle settings, select exact training files via a checklist, and watch loss logs and tokens/sec update **live** inside your browser!

### 2. 🛡️ Isolated Ubuntu PRoot Sandbox Environment (`n1bit/sandbox.py`)
- Allows your 1-bit AI to securely compile, run, play text games, draw ASCII charts, and execute code blocks!
- When **Sandbox Mode** is toggled on, any Python or Bash code blocks generated by the AI in chat are captured by a security guard.
- You can instantly execute those code snippets inside a real **Termux Ubuntu PRoot container** or secure simulation directory, outputting the `stdout`, `stderr`, and execution `exit_code` directly in your chat screen!

### 3. 🌋 Vulkan GPU Acceleration (`n1bit/vulkan_engine.py`)
- Supports running matrix math operations directly on integrated/mobile GPUs (like Qualcomm Adreno, ARM Mali, or Intel iGPUs) using cross-platform **Vulkan compute shaders**.
- If native Vulkan is supported by your PyTorch backend, the model will automatically move tensors to the GPU for incredibly fast execution.

### 4. 🪙 Strict Q1 Quantization & FP16 Acceleration
- **FP16 Half-Precision:** Continuous weights and activations are processed in `np.float16` to save up to **50% memory bandwidth** and fit natively inside ARM64 NEON registers.
- **Sign-Quantization:** Quantizes weights to `{-1, +1}` in a single, fast vectorized CPU instruction.
- **Auto-Save Resumable Training:** Saves loss progress and **complete Adam optimizer matrices** every 10 steps. If training is interrupted, restarting it will resume instantly from that exact step.

## ⚙️ Essential CLI Commands

### 1. Launching the Web UI Dashboard (Highly Recommended!)
Start the web server:
```bash
python app.py
```
Open your phone's web browser and go to:
👉 **`http://127.0.0.1:5000`**

### 2. Training Named Models & Selecting Datasets
To train a model with a specific name (e.g. `coder`) and select which of the 52 datasets to train on:
```bash
python train.py name coder
```
*You will be prompted with a beautiful numbered list of all your datasets. Type numbers separated by commas (e.g., `1, 3, 10, 15`) to choose!*

### 3. Infinite Pre-Training Loop
To run training indefinitely on a model (with active auto-saves and progress-logging):
```bash
python train.py inf
```
Or for a custom named model:
```bash
python train.py name coder inf
```

### 4. Interactive Console CLI (with Sandbox and Vulkan)
Start the terminal interface:
```bash
python interactive.py
```

## 🌋 Enabling Vulkan GPU Acceleration on Phone (Termux)

In standard Termux, GPU and Vulkan packages are not in the main repository, which leads to `Unable to locate package` errors. To enable Vulkan compute, follow these steps:

1. Enable the Termux User Repo (TUR) and X11 repo:
   ```bash
   pkg install tur-repo x11-repo -y
   ```
2. Update your package lists completely:
   ```bash
   pkg update && pkg upgrade -y
   ```
3. Install the Android Vulkan loader bridge, Vulkan tools, and shader compilers:
   ```bash
   pkg install vulkan-loader-android vulkan-tools vulkan-headers shaderc -y
   ```
4. Verify Vulkan is active on your phone's Adreno/Mali GPU:
   ```bash
   vulkaninfo
   ```
*Our 1-bit backend will automatically query Vulkan availability and accelerate matrices on GPU when active.*

## 🛡️ Setting up real Ubuntu PRoot Sandbox for AI

To let your AI run any script or command inside a real Ubuntu container on your phone:

1. Install PRoot Distro packages:
   ```bash
   pkg install proot proot-distro -y
   ```
2. Download and unpack Ubuntu:
   ```bash
   proot-distro install ubuntu
   ```
3. Turn on Option **`7`** inside `python interactive.py` to enable Sandbox execution!
